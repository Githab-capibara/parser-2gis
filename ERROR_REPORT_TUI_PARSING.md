Отчёт об ошибке
===============

Проблема:
---------
В TUI приложении parser-2gis обнаружены две критические ошибки:

1. **Ошибка "не выбраны города или категории" при запуске парсинга из главного меню:**
   - Кнопка "🚀 Запустить парсинг" в главном меню сразу открывала экран парсинга без проверки выбранных городов/категорий
   - В `parsing_screen.py` в методе `on_mount()` выполнялась проверка `self.app.selected_cities` и `self.app.selected_categories`, но эти списки были пустыми если пользователь не проходил через экраны выбора
   - Результат: парсинг не запускался, пользователь возвращался в меню без понятного объяснения причины

2. **Зависание при нажатии кнопки "Стоп":**
   - В методе `action_stop_parsing()` устанавливался флаг `self.app.running = False`, но это не гарантировало немедленную остановку парсера в потоке
   - Отсутствовала обработка случая когда парсинг ещё не начался (`_parsing_started = False`)
   - Результат: при нажатии кнопки "Стоп" до начала парсинга флаг остановки устанавливался, что могло привести к некорректному состоянию приложения

Причина:
--------
1. **Отсутствие валидации в main_menu.py:**
   - Обработчик `on_button_pressed()` для кнопки "start-parsing" не проверял наличие выбранных городов и категорий перед открытием экрана парсинга
   - Пользователь мог запустить парсинг без предварительной конфигурации

2. **Некорректная обработка состояния в parsing_screen.py:**
   - Метод `action_stop_parsing()` не проверял флаг `_parsing_started` перед установкой флага остановки
   - При вызове до начала парсинга метод просто возвращался без возврата в меню, оставляя пользователя на экране парсинга

3. **Недостаточная проверка флага остановки в app.py:**
   - В `progress_callback` присутствовала проверка `self._running`, но отсутствовала дополнительная проверка после обновления состояния
   - Отсутствовала проверка флага остановки после завершения работы парсера

Решение:
--------

### 1. Исправление main_menu.py

Добавлена проверка выбранных городов и категорий в обработчике кнопки "🚀 Запустить парсинг":

```python
if button_id == "start-parsing":
    # Проверка что выбраны города и категории
    if not self.app.selected_cities:
        self.app.notify("❌ Сначала выберите города в меню '📁 Выбрать города'", timeout=5)
        return
    if not self.app.selected_categories:
        self.app.notify("❌ Сначала выберите категории в меню '📂 Выбрать категории'", timeout=5)
        return
    # Только если оба списка не пустые - открывать экран парсинга
    self.app.push_screen("parsing")
```

**Изменения:**
- Добавлена проверка `selected_cities` - если список пуст, показывается уведомление и экран парсинга не открывается
- Добавлена проверка `selected_categories` - если список пуст, показывается уведомление и экран парсинга не открывается
- Экран парсинга открывается только если оба списка не пустые

### 2. Исправление parsing_screen.py

Изменён метод `action_stop_parsing()` для корректной обработки случая когда парсинг ещё не начался:

```python
def action_stop_parsing(self) -> None:
    # Защита от повторного вызова
    if self._stopping:
        return

    # Проверка что парсинг действительно запущен
    if not self._parsing_started:
        # Если парсинг ещё не начался - просто вернуться в меню без установки флага остановки
        self._add_log("[dim]Парсинг ещё не запущен, возврат в меню[/]")
        self.call_later(self._return_to_menu)
        return

    self._stopping = True
    self._add_log("[bold red]Остановка парсинга пользователем...[/]")
    self.app.running = False
    self.call_later(self._return_to_menu)
```

**Изменения:**
- Добавлена проверка `_parsing_started` перед установкой флага остановки
- Если парсинг ещё не начался - выполняется возврат в меню без установки флага `running = False`
- Добавлено логирование для отладки

### 3. Исправление app.py

Добавлены дополнительные проверки флага `_running` в методе `_run_parsing()`:

```python
def progress_callback(success: int, failed: int, filename: str) -> None:
    # Проверка флага остановки во время парсинга
    if not self._running:
        return

    category = filename.replace(".csv", "").split("_")[-1] if "_" in filename else ""
    self.update_state(
        success_count=success,
        error_count=failed,
        current_category=category,
        current_record=success + failed,
    )

    # Дополнительная проверка флага остановки после обновления состояния
    if not self._running:
        return
```

**Изменения:**
- Добавлена дополнительная проверка `_running` в `progress_callback` после обновления состояния
- Добавлена проверка `_running` после вызова `parser.run()` перед завершением парсинга
- Улучшен комментарий в `start_parsing()` для ясности порядка операций

Профилактика:
-------------

### Рекомендации для предотвращения подобных ошибок в будущем:

1. **Валидация входных данных:**
   - Всегда проверять наличие необходимых данных перед переходом к следующему экрану/действию
   - Показывать понятные сообщения об ошибках с указанием что нужно сделать

2. **Управление состоянием:**
   - Использовать флаги состояния для отслеживания текущего этапа работы
   - Проверять флаги состояния перед выполнением критических операций

3. **Потокобезопасность:**
   - При работе с потоками всегда проверять флаги остановки в callback'ах и циклах
   - Использовать `call_later()` для безопасного обновления UI из потоков

4. **Тестирование сценариев использования:**
   - Тестировать все возможные пути пользователя, включая "неправильные" сценарии
   - Особое внимание уделять обработке краевых случаев (пустые списки, ранняя остановка)

5. **Логирование:**
   - Добавлять логирование для отладки состояния приложения
   - Использовать разные уровни логирования (info, warning, error)
