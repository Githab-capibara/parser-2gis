Отчёт об ошибке
===============

Проблема:
---------
При запуске TUI интерфейса "./run.sh --tui" происходило полное зависание после выбора категории и нажатия кнопки "Далее". Невозможно было продолжить работу или вернуться в меню.

Симптомы:
- Пользователь выбирает города → переходит к выбору категорий
- Пользователь выбирает категории → нажимает "Далее"
- Интерфейс полностью зависает, никакие действия невозможны
- В логах отсутствуют сообщения об ошибках

Причина:
--------
Критическая ошибка в архитектуре переключения экранов TUI, вызвавшая циклический вызов и дублирование экранов в стеке:

1. **Файл `category_selector.py` (строка 229)**: При нажатии кнопки "Далее" вызывается `self.app.switch_screen("parsing")` для перехода на экран парсинга.

2. **Файл `parsing_screen.py` (строка 197)**: Метод `on_mount()` автоматически вызывается при монтировании экрана и выполняет `self.app.start_parsing(selected_cities, selected_cats)`.

3. **Файл `app.py` (строка 377, БЫЛО)**: Метод `start_parsing()` вызывал `self.push_screen("parsing")`, что создавало дублирование экрана в стеке.

4. **Файл `main_menu.py` (строка 168, БЫЛО)**: Кнопка "Запустить парсинг" также использовала `self.app.push_screen("parsing")`, усугубляя проблему.

**Циклический конфликт:**
```
category_selector.switch_screen("parsing")
    ↓
ParsingScreen.on_mount()
    ↓
app.start_parsing()
    ↓
app.push_screen("parsing") ← ПОВТОРНОЕ ОТКРЫТИЕ!
    ↓
ParsingScreen.on_mount() ← БЕСКОНЕЧНЫЙ ЦИКЛ
    ↓
... блокировка интерфейса ...
```

Решение:
--------
Исправлены два файла для устранения дублирования экранов:

### 1. Файл: `parser_2gis/tui_textual/app.py`

**Изменения в методе `start_parsing()` (строки 391-411):**

БЫЛО:
```python
def start_parsing(self, cities: list[dict], categories: list[dict]) -> None:
    # Проверка что данные выбраны перед запуском
    if not cities:
        self.notify_user("Ошибка: не выбраны города для парсинга!", level="error")
        return

    if not categories:
        self.notify_user("Ошибка: не выбраны категории для парсинга!", level="error")
        return

    # Сначала открываем экран парсинга, потом запускаем процесс
    self.push_screen("parsing")  # ← ПРОБЛЕМА!
    # Запуск парсинга в фоновом режиме
    self._run_parsing(cities, categories)
```

СТАЛО:
```python
def start_parsing(self, cities: list[dict], categories: list[dict]) -> None:
    # Проверка что данные выбраны перед запуском
    if not cities:
        self.notify_user("Ошибка: не выбраны города для парсинга!", level="error")
        return

    if not categories:
        self.notify_user("Ошибка: не выбраны категории для парсинга!", level="error")
        return

    # Экран парсинга уже открыт через switch_screen из category_selector
    # Запускаем только фоновый процесс парсинга
    # Примечание: push_screen НЕ вызывается здесь, чтобы избежать дублирования экранов
    self._run_parsing(cities, categories)
```

### 2. Файл: `parser_2gis/tui_textual/screens/main_menu.py`

**Изменения в методе `on_button_pressed()` (строки 167-171):**

БЫЛО:
```python
# Только если оба списка не пустые - открывать экран парсинга
self.app.push_screen("parsing")  # ← ПРОБЛЕМА!
```

СТАЛО:
```python
# Только если оба списка не пустые - открывать экран парсинга
# Используем switch_screen для замены текущего экрана вместо push_screen
# Это предотвращает накопление экранов в стеке и циклические вызовы
self.app.switch_screen("parsing")  # type: ignore
```

### Обоснование изменений:

- **`switch_screen()`** - заменяет текущий экран, не добавляя в стек. Используется когда нужно перейти на новый экран без возможности возврата через `pop_screen()`.
- **`push_screen()`** - добавляет экран в стек. Используется для временных экранов (настройки, диалоги).

Экран парсинга - конечный в цепочке навигации, поэтому должен открываться через `switch_screen()`.

Профилактика:
-------------
Для предотвращения подобных ошибок в будущем рекомендуется:

1. **Документировать навигацию экранов**: Создать схему навигации TUI с указанием, какие экраны открываются через `push_screen()`, а какие через `switch_screen()`.

2. **Добавить проверку в `ParsingScreen.on_mount()`**: Перед запуском парсинга проверять флаг `_parsing_started` для предотвращения повторного запуска:
   ```python
   def on_mount(self) -> None:
       if self._parsing_started:
           return  # Уже запущено
       # ... остальной код
   ```

3. **Разделить ответственность**: Метод `start_parsing()` в `app.py` должен только запускать фоновый процесс, но не управлять экранами. Управление экранами должно быть в ответственности экранов.

4. **Добавить логирование навигации**: Логировать все переключения экранов для отладки:
   ```python
   def switch_screen(self, name: str) -> None:
       self._file_logger.debug(f"Switching to screen: {name}")
       # ...原有代码
   ```

5. **Тестировать сценарии навигации**: Покрыть автотестами все сценарии:
   - Выбор городов → Выбор категорий → Парсинг
   - Главное меню → Запустить парсинг
   - Возврат из парсинга в меню

6. **Code Review для изменений навигации**: Любые изменения в переключении экранов должны проходить обязательное ревью с проверкой стека экранов.
