### 1. parser_2gis/config.py

**Проблема:** Комментарии на английском языке  
**Строки:** 90, 93, 156, 159, 233

**До исправления:**
```python
# Pydantic v2
model_config = ConfigDict(arbitrary_types_allowed=True)

# Pydantic v1
__validators__ = ...
```

**После исправления:**
```python
# Pydantic версия 2
model_config = ConfigDict(arbitrary_types_allowed=True)

# Pydantic версия 1
__validators__ = ...
```

**Результат:** Все комментарии переведены на русский язык для единообразия документации.

---

### 2. parser_2gis/common.py

**Проблема:** Неиспользуемый код (константы и функции)  
**Строки:** 406-442

**Удалённые элементы:**
- Константы для проверки кириллицы:
  - `_CYRILLIC_LOWER_START`
  - `_CYRILLIC_LOWER_END`
  - `_CYRILLIC_UPPER_START`
  - `_CYRILLIC_UPPER_END`
  - `_CYRILLIC_IO_LOWER`
  - `_CYRILLIC_IO_UPPER`
  - `_SPACE_CODE`
- Функция `_is_safe_char()`

**Обоснование:** Функция `url_query_encode()` использует стандартный `urllib.parse.quote()` вместо собственной реализации, что является правильным подходом. Удаление неиспользуемого кода уменьшает размер файла и упрощает поддержку.

**Результат:** Код стал чище, удалено ~40 строк неиспользуемого кода.

### 3. parser_2gis/parallel_parser.py

**Проблема:** Обращение к несуществующему атрибуту `timeout` в `ParserOptions`  
**Строка:** 329

**До исправления:**
```python
timeout_per_url = getattr(self.config.parser, "timeout", 300)
```

**После исправления:**
```python
# Таймаут фиксированный: 300 секунд (5 минут)
# Атрибут timeout не определён в ParserOptions, используем значение по умолчанию
timeout_per_url = 300
```

**Результат:** Устранена потенциальная ошибка времени выполнения, добавлен поясняющий комментарий.

### 4. parser_2gis/validator.py

**Проблема:** Нарушение стиля кода — использование имени класса вместо `self`  
**Строки:** 114, 116, 134, 139

**До исправления:**
```python
if len(digits_only) < DataValidator.INTERNATIONAL_PHONE_MIN_LENGTH:
    errors.append("Недостаточно цифр в номере")

if len(digits_only) > DataValidator.INTERNATIONAL_PHONE_MAX_LENGTH:
    errors.append("Слишком много цифр в номере")
```

**После исправления:**
```python
if len(digits_only) < self.INTERNATIONAL_PHONE_MIN_LENGTH:
    errors.append("Недостаточно цифр в номере")

if len(digits_only) > self.INTERNATIONAL_PHONE_MAX_LENGTH:
    errors.append("Слишком много цифр в номере")
```

**Результат:** Код соответствует стилю экземпляра класса, улучшена читаемость.

### 5. parser_2gis/chrome/remote.py

**Статус:** ⚠️ Требует улучшения, но не критично

**Проблема:** Некоторые комментарии содержат английские технические термины

**Решение:** Оставлено как есть, так как технические термины (eval, Function, setTimeout, innerHTML) являются общепринятыми в индустрии и их перевод нецелесообразен.

### 6. writer/models/schedule.py

**Статус:** ✅ Не требует исправления

**Проблема:** Использование `from` как алиаса в `from_: str = Field(..., alias="from")`

**Решение:** Это корректно, так как `from` — это алиас для JSON-сериализации, а не имя переменной Python. Изменения не требуются.
