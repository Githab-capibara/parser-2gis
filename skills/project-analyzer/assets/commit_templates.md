# Шаблоны сообщений коммитов для project-analyzer

## Формат сообщений

Используйте следующий формат для сообщений коммитов:

```
<тип>: <краткое описание>

<подробное описание (опционально)>
```

## Типы коммитов

### Исправления
- `fix:` - Исправление ошибки
- `fix(логика):` - Исправление логической ошибки
- `fix(синтаксис):` - Исправление синтаксической ошибки
- `fix(безопасность):` - Исправление уязвимости безопасности

### Улучшения
- `refactor:` - Рефакторинг кода
- `optimize:` - Оптимизация производительности
- `style:` - Исправление стиля кода
- `docs:` - Обновление документации

### Комментарии
- `comments:` - Перевод/добавление комментариев
- `i18n:` - Интернационализация (перевод на русский)

### Инфраструктура
- `chore:` - Вспомогательные изменения
- `config:` - Изменение конфигурации
- `github:` - Изменения, связанные с GitHub

## Примеры

### Простое исправление
```
fix: исправление ошибки деления на ноль в калькуляторе
```

### Исправление с деталями
```
fix(логика): исправление ошибки в расчёте скидки

- Добавлена проверка на отрицательные значения
- Исправлена формула расчёта итоговой суммы
- Добавлены юнит-тесты для граничных случаев
```

### Перевод комментариев
```
comments: перевод комментариев на русский язык

Все комментарии в main.py и utils.py переведены на русский
в соответствии с требованиями проекта.
```

### Рефакторинг
```
refactor: выделение повторяющейся логики в отдельные функции

- Создана функция validate_input() для проверки входных данных
- Создана функция format_output() для форматирования результата
- Устранено дублирование кода в модуле processor
```

### Оптимизация
```
optimize: улучшение производительности обработки данных

- Замена цикла на list comprehension
- Кэширование результатов дорогостоящих вычислений
- Оптимизация работы с файлами через контекстный менеджер
```

## Чек-лист перед коммитом

- [ ] Все изменения протестированы
- [ ] Комментарии переведены на русский
- [ ] Код соответствует стандартам проекта
- [ ] Сообщение коммита понятное и информативное
- [ ] Файл QWEN.md не включён в коммит

## Запрещённые практики

❌ Не используйте общие сообщения:
- "fix bug"
- "update code"
- "changes"
- "minor fixes"

✅ Используйте конкретные сообщения:
- "fix: исправление ошибки округления в финансовом модуле"
- "refactor: упрощение условия в функции authenticate"
- "comments: перевод документации на русский язык"
