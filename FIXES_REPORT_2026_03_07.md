## Исправленные проблемы

### 1. parser_2gis/main.py

**Проблема:** Избыточная проверка параметров командной строки в режиме `categories_mode`.

**Описание:** В функции `main()` выполнялась проверка обязательных параметров (`output_path`, `format`) даже после завершения параллельного парсинга в режиме `categories_mode`, что приводило к излишним сообщениям об ошибках.

**Исправление:**
```python
# В режиме categories_mode парсинг уже завершён выше, пропускаем проверки
if not categories_mode:
    # Обычный режим - требуем все обязательные параметры
    if not urls and not (hasattr(args, 'cities') and args.cities):
        logger.error('Не указан источник URL. Используйте -i/--url или --cities')
        sys.exit(1)
    
    if not args.output_path:
        logger.error('Не указан путь к выходному файлу. Используйте -o/--output-path')
        sys.exit(1)
    
    if not args.format:
        logger.error('Не указан формат выходного файла. Используйте -f/--format')
        sys.exit(1)
```

**Влияние:** Устранены ложные ошибки при использовании режима параллельного парсинга по категориям.

### 2. parser_2gis/chrome/browser.py

**Проблема:** Потенциальная ошибка при использовании Chrome с неуказанным `memory_limit`.

**Описание:** Параметр `memory_limit` напрямую использовался в командной строке без проверки на `None`, что могло привести к ошибкам или некорректной работе браузера.

**Исправление:**
```python
# Валидация memory_limit перед формированием команды
memory_limit = chrome_options.memory_limit if chrome_options.memory_limit is not None else 2048

# Формирование команды запуска
self._chrome_cmd = [
    # ...
    f'--js-flags=--expose-gc --max-old-space-size={memory_limit}',
    # ...
]
```

**Влияние:** Добавлен механизм защиты со значением по умолчанию 2048 МБ (2 ГБ).

### 3. parser_2gis/writer/writers/csv_writer.py

**Проблема:** Отсутствие явного указания кодировки при чтении/записи CSV файлов.

**Описание:** Функции `_remove_empty_columns()` и `_remove_duplicates()` использовали методы `_open_file()` без явного указания кодировки, что могло привести к проблемам с кириллицей на разных системах.

**Исправление:**
```python
# Первый проход: подсчёт непустых значений в сложных колонках
with self._open_file(self._file_path, 'r', encoding='utf-8-sig') as f_csv:
    # ...

# Чтение исходного файла и запись нового без дубликатов
with self._open_file(self._file_path, 'r', encoding='utf-8-sig') as f_csv, \
     self._open_file(tmp_csv_name, 'w', encoding='utf-8', newline='') as f_tmp_csv:
    # ...
```

**Влияние:** Гарантирована корректная работа с кириллическими символами во всех операционных системах.
