# Отчёт об исправлениях: parallel_parser.py

## Дата: 2026-03-06

## Исправленные критические проблемы

### 1. Отсутствует валидация входных данных ✅
**Место:** `__init__` метод (строки ~43-75)

**Исправления:**
- Добавлена проверка `cities` на пустоту: `if not cities: raise ValueError("Список городов не может быть пустым")`
- Добавлена проверка `categories` на пустоту: `if not categories: raise ValueError("Список категорий не может быть пустым")`
- Добавлено ограничение `max_workers` (1-20): `if not 1 <= max_workers <= 20: raise ValueError("max_workers должен быть от 1 до 20")`
- Добавлена проверка `output_dir` на существование и права на запись
- При несуществовании директории - попытка создания с последующей проверкой прав

### 2. Race condition в parse_single_url ✅
**Место:** `parse_single_url` метод (строки ~127-185)

**Исправления:**
- Используется уникальное временное имя файла с UUID: `{city}_{category}_{uuid}.tmp`
- Запись происходит во временный файл
- После успешного парсинга файл атомарно переименовывается в целевое имя через `temp_filepath.replace(filepath)`
- При ошибке временный файл удаляется

### 3. merge_csv_files удаляет файлы prematurely ✅
**Место:** `merge_csv_files` метод (строки ~187-272)

**Исправления:**
- Создан список `files_to_delete` для накопления файлов
- В цикле файлы только добавляются в список, но не удаляются
- ТОЛЬКО ПОСЛЕ успешного завершения цикла удаляются все файлы из списка
- Добавлено логирование процесса удаления
- При ошибке в цикле файлы не удаляются (логирование предупреждения)

### 4. Отсутствует таймаут на парсинг одного URL ✅
**Место:** `run` метод (строки ~274-340)

**Исправления:**
- Получение таймаута из конфигурации: `timeout_per_url = getattr(self.config.parser, 'timeout', 300)`
- Использование `future.result(timeout=timeout_per_url)` для установки таймаута
- Обработка `TimeoutError` с логированием
- Логирование значения таймаута при старте

### 5. Нет проверки на существование output_dir и прав на запись ✅
**Место:** `__init__` метод (строки ~58-75)

**Исправления:**
- Проверка `if self.output_dir.exists()`: если существует - проверка `is_dir()` и `os.access(..., os.W_OK)`
- Если не существует - попытка создания с `mkdir(parents=True, exist_ok=True)`
- Последующая проверка прав на запись
- Генерация `ValueError` с подробным сообщением при любой проблеме
