## Исправленные проблемы

### 1. Логика (строки 68-78) - _get_links
**Проблема:** `wait_until_finished` с `throw_exception=False`, но возвращаемое значение не проверяется на None.

**Исправление:**
- Изменён тип возвращаемого значения на `Optional[list[DOMNode]]`
- Добавлена проверка: если `links` пустой, возвращается `None`
- Явная проверка на `None` в вызывающем коде

### 2. Логика (строки 168-172) - get_unique_links
**Проблема:** Декорированная функция может вернуть `[]`, что интерпретируется как успех.

**Исправление:**
- Изменён тип возвращаемого значения на `Optional[list[DOMNode]]`
- Возвращает `None` вместо `[]` при повторе ссылок
- Добавлена проверка `_get_links()` на `None`

### 3. Утечка ресурсов (строки 145-230) - parse цикл
**Проблема:** При exception `chrome_remote` может не закрыться.

**Исправление:**
- Обёрнуто в `try/except/finally` блок
- `finally` гарантирует вызов `clear_requests()`
- При критической ошибке логирование и перевыброс исключения

### 4. Логика (строки 189-197) - 3 попытки
**Проблема:** Нет таймаута между попытками получения ответа.

**Исправление:**
- Добавлена задержка `0.5` секунды между попытками
- Задержка применяется только если это не последняя попытка

### 5. Логика (строки 213-214) - clear_requests
**Проблема:** Вызывается после каждой страницы, может удалить нужные запросы.

**Исправление:**
- Перемещено в конец цикла, ПОСЛЕ перехода на следующую страницу
- Дополнительный вызов в `finally` для гарантии очистки
