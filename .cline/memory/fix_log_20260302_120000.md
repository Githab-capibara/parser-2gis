## Исправленные проблемы

### 1. Потокобезопасность (строки 297-303)
**Проблема:** `parsing_thread` проверяется без блокировки в основном цикле  
**Решение:** Добавлен `threading.Lock()` для доступа к `parsing_thread`. Функция `parsing_thread_running()` теперь использует блокировку.

### 2. Потокобезопасность (строки 331-336)
**Проблема:** Остановка потока в event handler без гарантии что поток не в критической секции  
**Решение:** Обёрнуто в `with parsing_thread_lock:` с добавлением флага `stop_flag` для polling.

### 3. Логика (строки 318-320)
**Проблема:** `gui_urls_editor` вызывается без проверки возвращаемого значения на None  
**Решение:** Добавлена проверка `if ret_urls is not None and isinstance(ret_urls, list):`

### 4. Утечка ресурсов (строки 379-381)
**Проблема:** При закрытии окна поток может не завершиться корректно  
**Решение:** Добавлен `join(timeout=5.0)` при закрытии окна с предупреждением если поток не завершился.

### 5. Логика (строки 114-115)
**Проблема:** `setup_cli_logger` вызывается до создания GUI logger queue - логи могут потеряться  
**Решение:** `setup_gui_logger` с созданием `log_queue` перемещено перед `setup_cli_logger`.
