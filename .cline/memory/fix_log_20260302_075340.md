# Отчёт об исправлении критических проблем QC

**Дата:** 2026-03-02 07:53:40  
**Файлы:** parser_2gis/writer/writers/*.py

## file_writer.py

### Исправленные проблемы:
1. **Логика (строки 44-76)**: `_check_catalog_doc` ловит только `KeyError`, но может возникнуть `TypeError`, `AttributeError`
   - **Изменение:** `except KeyError:` → `except (KeyError, TypeError, AttributeError):`

2. **Утечка ресурсов (строки 79-84)**: `__exit__` закрывает файл, но если `_file` не был создан - будет `AttributeError`
   - **Изменение:** Добавлена проверка `if hasattr(self, '_file'):` перед закрытием

---

## csv_writer.py

### Исправленные проблемы:
1. **Утечка ресурсов (строки 108-110, 120-122)**: Файлы открываются в контексте, но при exception между ними могут остаться открытыми
   - **Изменение:** Использован `contextlib.ExitStack` для гарантии закрытия файлов в `_remove_empty_columns` и `_remove_duplicates`

2. **Логика (строки 131-144)**: `_remove_duplicates` сравнивает полные строки включая newline
   - **Изменение:** Добавлена нормализация строк: `normalized_line = line.rstrip('\r\n')`

3. **Безопасность (строка 168)**: `logger.error(error_str)` может раскрыть структуру данных API
   - **Изменение:** Sanitized вывод — вместо полного документа выводятся только тип и ID

## xlsx_writer.py

### Исправленные проблемы:
1. **Утечка ресурсов (строки 18-32)**: При ошибке конвертации временный файл не удаляется
   - **Изменение:** Добавлен `try/except` с удалением временного файла при ошибке

2. **Логика (строка 25)**: `Workbook` без явного указания `constant_memory=True` может потреблять много RAM
   - **Изменение:** Добавлен параметр `{'constant_memory': True}` в конструктор `Workbook`

---

## json_writer.py

### Исправленные проблемы:
1. **Логика (строки 14-19)**: JSON массив формируется вручную - риск невалидного JSON при exception
   - **Изменение:** Используется `json.dump()` со списком элементов вместо ручной записи `[`, `,`, `]`
   - Элементы собираются в список `_items` и записываются одним вызовом в `__exit__`
