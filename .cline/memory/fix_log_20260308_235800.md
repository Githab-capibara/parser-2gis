## Исправленные проблемы

### 1. remote-allow-origins (строка 67)
- **Проблема:** `--remote-allow-origins=http://localhost` не защищает от всех портов
- **Решение:** Изменено на `--remote-allow-origins=http://localhost:*`

### 2. Проверка на WSL (строки 106-108)
- **Проблема:** Проверка `os.name != "nt"` не учитывает WSL
- **Решение:** Добавлена проверка на WSL через `platform.uname().release`
- **Код:**
```python
is_wsl = "microsoft" in platform.uname().release.lower()
if not is_wsl:
    logger.warning("Файл браузера не имеет прав на выполнение: %s", binary_path)
```

### 3. Импорт TimeoutExpired (строки 177-179)
- **Проблема:** `TimeoutExpired` не был импортирован
- **Решение:** Добавлен импорт `from subprocess import TimeoutExpired`
- **Примечание:** В коде используется `subprocess.TimeoutExpired`, что также корректно

### 4. Разделение js-flags (строки 71-73)
- **Проблема:** Один флаг для двух опций мог не работать
- **Решение:** Разделено на два флага:
```python
"--js-flags=--expose-gc",
f"--max-old-space-size={memory_limit}",
```

### 5. cleanup_marker в temp директории (строки 154-158)
- **Проблема:** Создание маркера в родительской директории профиля могло вызвать PermissionError
- **Решение:** Маркер создаётся в `tempfile.gettempdir()`
```python
marker_file = os.path.join(tempfile.gettempdir(), ".cleanup_marker")
```

### 6. Декоратор _delete_profile (строки 133-161)
- **Статус:** Оставлено без изменений
- **Причина:** Декоратор `@wait_until_finished` корректно обрабатывает return value
