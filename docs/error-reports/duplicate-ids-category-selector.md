Отчёт об ошибке
===============

Проблема:
---------
При фильтрации категорий в методе on_input_changed() возникала ошибка DuplicateIds от фреймворка Textual.

Сценарий ошибки:
1. Пользователь вводит текст в поле поиска категорий
2. Вызывается on_input_changed(), который фильтрует категории
3. Вызывается _populate_categories() для обновления списка
4. Старый код делал:
   - container.remove_children() - удаляет виджеты из контейнера
   - self._checkboxes.clear() - очищает список ссылок
   - Создавались новые Checkbox с теми же ID (например, 'category-0')
   - container.mount(checkbox) - возникала ошибка DuplicateIds

Текст ошибки (типичный):
"Duplicate ID: category-0 - Multiple widgets with the same ID are not allowed"

Причина:
--------
Метод remove_children() удаляет виджеты только из родительского контейнера, 
но НЕ удаляет их из реестра приложения Textual. Когда mount() пытается добавить 
новый виджет с тем же ID, Textual обнаруживает конфликт в своём внутреннем реестре 
и выбрасывает исключение DuplicateIds.

Корневая проблема: несоответствие между жизненным циклом виджетов и управлением 
реестром Textual. Для корректного удаления виджета необходимо явно вызвать его 
метод remove(), который unregister'ит виджет из приложения.

Решение:
--------
Изменён метод _populate_categories() в файле:
parser_2gis/tui_textual/screens/category_selector.py

Было:
```python
container = self.query_one("#category-list", ScrollableContainer)
container.remove_children()
self._checkboxes.clear()
```

Стало:
```python
container = self.query_one("#category-list", ScrollableContainer)

# Явно удаляем каждый виджет через remove() — это критически важно!
# remove_children() не удаляет виджеты из реестра приложения Textual,
# поэтому при повторном mount() с тем же ID возникает ошибка DuplicateIds.
# Метод remove() корректно unregister'ит виджет из приложения.
for checkbox in self._checkboxes:
    checkbox.remove()
self._checkboxes.clear()
```

Изменения гарантируют:
1. Каждый Checkbox корректно удаляется из реестра Textual
2. При создании новых виджетов с теми же ID конфликтов не возникает
3. Фильтрация категорий работает без ошибок
4. Все существующие функции (выбор категорий, кнопки) работают корректно

Профилактика:
-------------
1. При работе с динамическими виджетами в Textual всегда использовать remove() 
   для каждого виджета перед очисткой контейнера

2. Избегать remove_children() когда планируется повторное использование ID виджетов

3. Рассмотреть альтернативные подходы для будущих реализаций:
   - Использовать реактивный механизм Textual с compose()
   - Не использовать ID для динамических виджетов (использовать data-атрибуты)
   - Использовать уникальный ID для каждой инстанции виджета

4. Добавить unit-тесты для сценариев фильтрации и обновления динамических списков

5. При ревью кода обращать внимание на жизненный цикл виджетов в Textual
