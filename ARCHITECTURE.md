# Архитектура проекта parser-2gis

**Версия:** 1.2.2  
**Дата обновления:** 2026-03-23

## Общая схема проекта

```
parser_2gis/
├── chrome/              # Модуль работы с браузером Chrome
├── cli/                 # CLI интерфейс
├── data/                # Данные (города, категории, рубрики)
├── logger/              # Модуль логирования
├── parallel/            # Модуль параллельной обработки
├── parser/              # Модуль парсинга данных
├── runner/              # Модуль запуска парсера
├── tui_textual/         # TUI интерфейс на Textual
├── writer/              # Модуль записи данных
├── cache.py             # Кэширование на SQLite
├── common.py            # Общие утилиты
├── config.py            # Конфигурация через Pydantic
├── constants.py         # Централизованные константы
├── exceptions.py        # Иерархия исключений
├── main.py              # CLI точка входа
├── parallel_parser.py   # Параллельный парсинг
├── parallel_helpers.py  # Вспомогательные классы параллелизма
├── paths.py             # Управление путями
├── signal_handler.py    # Обработчик сигналов
├── statistics.py        # Статистика работы
├── validation.py        # Валидация данных
└── version.py           # Версия пакета
```

## Основные модули

### chrome/
Управление браузером Chrome через Chrome DevTools Protocol.

**Ключевые классы:**
- `ChromeBrowser` - Управление экземпляром браузера
- `ChromeRemote` - Работа с DevTools Protocol
- `FileLogger` - Логирование в файл

**Исключения:**
- `ChromeException` - Базовое исключение
- `ChromeRuntimeException` - Ошибка времени выполнения
- `ChromePathNotFound` - Браузер не найден

### parser/
Парсинг данных с 2GIS.

**Ключевые классы:**
- `BaseParser` - Абстрактный базовый класс парсера
- `FirmParser` - Парсинг организаций
- `InBuildingParser` - Парсинг внутри зданий
- `MainParser` - Главный парсер

**Функции:**
- `get_parser()` - Фабричный метод создания парсера

### writer/
Запись данных в различные форматы.

**Классы:**
- `FileWriter` - Базовый класс записи
- `CSVWriter` - Запись в CSV
- `XLSXWriter` - Запись в XLSX
- `JSONWriter` - Запись в JSON

**Функции:**
- `get_writer()` - Фабричный метод создания writer

### parallel/
Параллельная обработка данных.

**Классы:**
- `ParallelCityParser` - Параллельный парсинг городов
- `FileMerger` - Слияние CSV файлов
- `ProgressTracker` - Отслеживание прогресса

### cache/
Кэширование результатов на SQLite.

**Классы:**
- `CacheManager` - Менеджер кэша
- `_ConnectionPool` - Пул соединений

**Особенности:**
- TTL 24 часа
- Connection pool (5-20 соединений)
- WAL режим для производительности

### logger/
Логирование работы приложения.

**Компоненты:**
- `setup_cli_logger()` - Настройка CLI логгера
- `FileLogger` - Логирование в файл
- `VisualLogger` - Визуальный прогресс

## Паттерны проектирования

### Фабричный метод (Factory Method)
Используется для создания парсеров и writers:
- `get_parser()` в `parser/factory.py`
- `get_writer()` в `writer/factory.py`

### Одиночка (Singleton)
- `CacheManager` - единый экземпляр для кэширования
- `SignalHandler` - глобальный обработчик сигналов

### Стратегия (Strategy)
- Различные парсеры (`FirmParser`, `InBuildingParser`, `MainParser`)
- Различные writers (`CSVWriter`, `XLSXWriter`, `JSONWriter`)

### Контекстный менеджер (Context Manager)
- `FileMerger` - гарантия очистки временных файлов
- `ChromeBrowser` - гарантия закрытия браузера

### Пул объектов (Object Pool)
- `_ConnectionPool` в `cache.py` - пул SQLite соединений

## Иерархия исключений

```
Exception
└── BaseContextualException (базовое исключение с контекстом)
    ├── ChromeException
    │   ├── ChromeRuntimeException
    │   ├── ChromeUserAbortException
    │   └── ChromePathNotFound
    ├── ParserException
    └── WriterUnknownFileFormat
```

**Особенности BaseContextualException:**
- Автоматическое извлечение имени функции
- Номер строки где произошла ошибка
- Имя файла
- Полная трассировка стека

## Константы

Все константы централизованы в `constants.py`:

### Безопасность данных
- `MAX_DATA_DEPTH = 15` - Максимальная глубина вложенности
- `MAX_STRING_LENGTH = 10000` - Максимальная длина строки
- `MAX_DATA_SIZE = 10MB` - Максимальный размер данных

### Параллельный парсинг
- `MIN_WORKERS = 1` - Минимум потоков
- `MAX_WORKERS = 20` - Максимум потоков
- `DEFAULT_TIMEOUT = 300` - Таймаут по умолчанию

### Буферизация
- `DEFAULT_BUFFER_SIZE = 512KB` - Размер буфера
- `MERGE_BUFFER_SIZE = 128KB` - Буфер для слияния
- `CSV_BATCH_SIZE = 1000` - Пакет строк для CSV

## Производительность

### Оптимизации
- lru_cache для часто вызываемых функций
- Компилированные regex паттерны
- Пакетная запись данных (до 500 записей)
- mmap для больших файлов (>10MB)
- Connection pooling для SQLite

### Масштабирование
- До 20 параллельных потоков
- Динамический размер пула соединений
- Балансировка нагрузки между потоками

## Расширяемость

### Добавление нового парсера
1. Создать класс наследуясь от `BaseParser`
2. Реализовать абстрактные методы `parse()` и `get_stats()`
3. Зарегистрировать в фабрике парсеров

### Добавление нового формата вывода
1. Создать класс наследуясь от `FileWriter`
2. Реализовать метод `write()`
3. Зарегистрировать в фабрике writers

### Добавление новой константы
1. Добавить константу в `constants.py`
2. Добавить в `__all__` для экспорта
3. Импортировать в нужных модулях
