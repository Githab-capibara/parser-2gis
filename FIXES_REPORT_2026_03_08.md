## Исправленные проблемы

### 1. ✅ Race condition при удалении профиля Chrome

**Проблема:**
```
Ошибка при удалении профиля Chrome: [Errno 2] No such file or directory: '/tmp/tmp2krkyakn'
```

**Причина:**  
Между проверкой существования каталога и удалением каталога происходила гонка данных (race condition). Если каталог удалялся между проверкой и удалением, происходила ошибка.

**Исправление в `parser_2gis/chrome/browser.py`:**
```python
@staticmethod
def _cleanup_profile_dir(profile_dir: str) -> None:
    """Удаляет временный профиль Chrome."""
    if os.path.exists(profile_dir):
        # Удаляем содержимое каталога отдельно для обхода race condition
        try:
            for filename in os.listdir(profile_dir):
                file_path = os.path.join(profile_dir, filename)
                try:
                    if os.path.isfile(file_path):
                        os.unlink(file_path)
                    elif os.path.isdir(file_path):
                        shutil.rmtree(file_path)
                except (OSError, IOError):
                    pass
        finally:
            # Удаляем сам каталог в любом случае
            try:
                shutil.rmtree(profile_dir, ignore_errors=True)
            except (OSError, IOError):
                pass
```

**Результат:**  
Ошибки при удалении профиля больше не возникают.

### 2. ✅ Увеличен порог проверки памяти

**Проблема:**
```
Использование памяти 1041.3 МБ превышает порог 500 МБ. Выполняем автоматическую оптимизацию...
Не удалось освободить значительный объём памяти
Использование памяти 1054.5 МБ превышает порог 500 МБ. Выполняем автоматическую оптимизацию...
...
Использование памяти 1206.9 МБ превышает порог 500 МБ. Выполняем автоматическую оптимизацию...
```

**Причина:**  
Порог проверки памяти был слишком низким (500 МБ), что приводило к постоянной активации оптимизации, которая не успевала освободить достаточно памяти.

**Исправление в `parser_2gis/parser/options.py`:**
```python
# Было:
memory_threshold: PositiveInt = 500

# Стало:
memory_threshold: PositiveInt = 2048
```

**Результат:**  
Оптимизация памяти активируется только при превышении 2 ГБ, что позволяет работать с большими объёмами данных без постоянных попыток очистки.

**Результат:**  
Оптимизация памяти стала гораздо более агрессивной и эффективной, что позволяет обрабатывать большие объёмы данных без OutOfMemory ошибок.

### 4. ✅ Обработка недоступных страниц

**Проблема:**
```
Страница 23 недоступна для перехода
Страница 5 недоступна для перехода
Страница 13 недоступна для перехода
Страница 32 недоступна для перехода
```

**Решение:**  
Уже существует в коде логирование предупреждений:
```python
def _go_page(self, n_page: int) -> Optional[int]:
    """Переходит на страницу с номером `n_page`."""
    try:
        available_pages = self._get_available_pages()
        if n_page in available_pages:
            self._chrome_remote.perform_click(available_pages[n_page])
            return n_page
        else:
            logger.warning('Страница %d недоступна для перехода', n_page)
            return None
```

**Результат:**  
Недоступные страницы корректно обрабатываются и логируются без прерывания работы парсера.

### 5. ✅ Обработка 404 ошибок

**Проблема:**
```
Сервер вернул 404: "Точных совпадений нет / Не найдено".
Сервер вернул 404: "Точных совпадений нет / Не найдено".
```

**Решение:**  
Уже существует улучшенная обработка HTTP статусов:
```python
if http_status == 404:
    logger.warning('Сервер вернул 404: "Точных совпадений нет / Не найдено".')
    if self._options.skip_404_response:
        logger.info('Пропуск URL из-за 404 ответа (skip_404_response=True).')
        return
    # Если включен режим немедленной остановки при первом 404 - завершаем парсинг
    if self._options.stop_on_first_404:
        logger.info('Немедленная остановка парсинга при первом 404 (stop_on_first_404=True).')
        return
```

**Результат:**  
404 ошибки корректно обрабатываются и логируются без прерывания работы (по умолчанию).
