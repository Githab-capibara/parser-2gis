## 📬 Сообщение об уязвимостях

### Как сообщить об уязвимости

Если вы обнаружили уязвимость в Parser2GIS, пожалуйста:

1. **НЕ создавайте публичный Issue** на GitHub
2. Отправьте отчёт на электронную почту или в GitHub Security Advisory
3. Предоставьте подробное описание уязвимости
4. Приложите шаги для воспроизведения (если возможно)
5. Укажите версию Parser2GIS и окружение

### Что включать в отчёт

- **Тип уязвимости** (XSS, RCE, инъекция, утечка данных и т.д.)
- **Затронутые компоненты** (файлы, функции, классы)
- **Версия Parser2GIS**
- **Версия Python и ОС**
- **Шаги для воспроизведения**
- **Потенциальный ущерб**
- **Рекомендации по исправлению** (если есть)

### Время реакции

- **Критические уязвимости**: 24-48 часов
- **Высокие уязвимости**: 3-5 дней
- **Средние уязвимости**: 1-2 недели
- **Низкие уязвимости**: По возможности

## 🔒 Рекомендации по безопасности

### 1. Управление секретами

#### ❌ НЕЛЬЗЯ
```bash
# Никогда не храните токены в коде
GITHUB_TOKEN="ghp_..."  # В коде или конфиге
```

#### ✅ МОЖНО
```bash
# Используйте переменные окружения
export GITHUB_TOKEN="ghp_..."

# Или файл .env (добавлен в .gitignore)
echo "GITHUB_TOKEN=ghp_..." >> .env
```

### 2. Безопасность зависимостей

#### Регулярно обновляйте зависимости
```bash
# Проверка уязвимостей
pip audit

# Обновление зависимостей
pip install --upgrade parser-2gis

# Проверка установленных пакетов
pip list --outdated
```

#### Минимальные необходимые разрешения
```bash
# Устанавливайте только необходимые зависимости
pip install parser-2gis  # Базовая версия
pip install parser-2gis[dev]  # Только для разработки
```

### 3. Безопасность браузера

#### Изоляция браузера
```bash
# Запускайте в headless режиме на серверах
parser-2gis --chrome.headless yes ...

# Отключайте изображения для экономии трафика
parser-2gis --chrome.disable-images yes ...

# Устанавливайте лимит памяти
parser-2gis --chrome.memory-limit 2048 ...
```

#### Не используйте для чувствительных данных
- ⚠️ **Не вводите** логины/пароли от аккаунтов 2GIS
- ⚠️ **Не сохраняйте** cookies и сессионные данные
- ⚠️ **Не используйте** с корпоративными прокси без проверки

### 4. Безопасность файлов

#### Защита выходных данных
```bash
# Устанавливайте безопасные права на файлы
chmod 600 output/*.csv
chmod 600 output/*.xlsx
chmod 600 output/*.json

# Шифруйте чувствительные данные
gpg -c output/data.csv
```

#### Очистка временных файлов
```bash
# Регулярно очищайте кэш
rm -rf /tmp/parser_cache/*

# Очищайте логи после анализа
rm logs/parser_*.log
```

### 5. Сетевая безопасность

#### Ограничьте доступ
```bash
# Используйте фаервол для ограничения доступа
ufw deny from any to any port 9222  # Chrome DevTools порт

# Разрешите только локальный доступ
ufw allow from 127.0.0.1 to any port 9222
```

#### Мониторинг сетевой активности
```bash
# Проверка активных соединений
netstat -tulpn | grep chrome

# Мониторинг трафика
tcpdump -i any port 80 or port 443
```

## ⚙️ Безопасная конфигурация

### Минимальная безопасная конфигурация

```json
{
  "chrome": {
    "headless": true,
    "disable_images": true,
    "memory_limit": 2048,
    "silent_browser": true
  },
  "parser": {
    "max_records": 1000,
    "stop_on_first_404": true,
    "max_consecutive_empty_pages": 5,
    "max_retries": 3,
    "retry_on_network_errors": true,
    "memory_threshold": 2048
  },
  "writer": {
    "encoding": "utf-8-sig",
    "remove_duplicates": true,
    "remove_empty_columns": true
  }
}
```

### Запуск с безопасными настройками

```bash
parser-2gis \
  --cities moscow spb \
  --categories-mode \
  --parallel-workers 5 \
  --chrome.headless yes \
  --chrome.disable-images yes \
  --chrome.memory-limit 2048 \
  --parser.stop-on-first-404 yes \
  --parser.max-consecutive-empty-pages 5 \
  --parser.max-retries 3 \
  --parser.retry-on-network-errors yes \
  --parser.memory-threshold 2048 \
  -o output/ \
  -f csv
```
