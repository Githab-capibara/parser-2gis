## Сообщение об уязвимости

Мы серьёзно относимся к безопасности Parser2GIS и благодарим за ответственное сообщение об уязвимостях.

### Как сообщить

**⚠️ ВАЖНО:** Не создавайте публичные issues для сообщений об уязвимостях безопасности!

Для сообщения об уязвимости:

1. **Email:** Отправьте письмо на `security@parser-2gis.local` (настройте через GitHub Private Vulnerability Reporting)

2. **GitHub Private Vulnerability Reporting:**
   - Перейдите на вкладку [Security](https://github.com/Githab-capibara/parser-2gis/security)
   - Нажмите "Report a vulnerability"
   - Заполните форму с подробным описанием

3. **Что включить в отчёт:**
   - Тип уязвимости (SQL injection, XSS, RCE, и т.д.)
   - Шаги для воспроизведения
   - Версия Parser2GIS
   - Возможные последствия
   - Предложения по исправлению (если есть)

## Исправленные уязвимости (v2.1.6)

### Критические (Critical)

#### 1. SQL Injection в CacheManager
- **CVE:** Назначается при публикации
- **Severity:** Critical (CVSS 9.8)
- **Описание:** Возможность SQL injection через невалидированные хеши
- **Исправление:** Добавлена строгая валидация SHA256 хешей
- **Версии:** < 2.1.6
- **Благодарность:** Команда безопасности

#### 2. XSS через выполнение JavaScript
- **CVE:** Назначается при публикации
- **Severity:** Critical (CVSS 9.1)
- **Описание:** Выполнение произвольного JavaScript через Chrome
- **Исправление:** Валидация и логирование всех скриптов
- **Версии:** < 2.1.6
- **Благодарность:** Команда безопасности

### Высокие (High)

#### 3. SSRF через валидацию URL
- **CVE:** Назначается при публикации
- **Severity:** High (CVSS 8.2)
- **Описание:** Возможность атаки на внутренние сервисы
- **Исправление:** Блокировка localhost и private IP
- **Версии:** < 2.1.6
- **Благодарность:** Команда безопасности

#### 4. Race Condition в именах файлов
- **CVE:** Назначается при публикации
- **Severity:** High (CVSS 7.5)
- **Описание:** Коллизии имён временных файлов
- **Исправление:** Добавлен PID и retry логика
- **Версии:** < 2.1.6
- **Благодарность:** Команда безопасности

### Средние (Medium)

#### 5. Утечка файловых дескрипторов
- **Severity:** Medium (CVSS 5.3)
- **Описание:** Утечка ресурсов при ошибках
- **Исправление:** Гарантированная очистка в finally
- **Версии:** < 2.1.6

#### 6. Deadlock в конфигурации
- **Severity:** Medium (CVSS 5.0)
- **Описание:** Взаимная блокировка при merge
- **Исправление:** Использование RLock и timeout
- **Версии:** < 2.1.6

### Низкие (Low)

#### 7. UnicodeDecodeError
- **Severity:** Low (CVSS 3.1)
- **Описание:** Ошибки кодировки при чтении кэша
- **Исправление:** Fallback на альтернативные кодировки
- **Версии:** < 2.1.6

## Рекомендации по безопасности

### Для пользователей

1. **Обновляйтесь регулярно:**
   ```bash
   pip install --upgrade parser-2gis
   ```

2. **Используйте виртуальное окружение:**
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   pip install parser-2gis
   ```

3. **Настройте переменные окружения:**
   ```bash
   export GITHUB_TOKEN="your_token"
   export PARSER_LOG_LEVEL="INFO"
   ```

4. **Используйте разумные лимиты:**
   ```bash
   parser-2gis --parser.max-records 1000
   ```

5. **Включайте headless режим на серверах:**
   ```bash
   parser-2gis --chrome.headless yes
   ```

### Для разработчиков

1. **Проверяйте зависимости:**
   ```bash
   pip audit
   pip-review --auto
   ```

2. **Запускайте тесты:**
   ```bash
   pytest tests/ -v
   ```

3. **Используйте pre-commit хуки:**
   ```bash
   pre-commit install
   pre-commit run --all-files
   ```

4. **Сканируйте код:**
   ```bash
   bandit -r parser_2gis/
   safety check
   ```
