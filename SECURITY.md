## Сообщение об уязвимости

Мы серьёзно относимся к безопасности Parser2GIS и благодарим за ответственное сообщение об уязвимостях.

### ⚠️ Важно

**Не создавайте публичные issues для сообщений об уязвимостях безопасности!**

### 📬 Как сообщить

**1. GitHub Private Vulnerability Reporting (рекомендуется):**

```
1. Перейдите на вкладку Security
2. Нажмите "Report a vulnerability"
3. Заполните форму с подробным описанием
```

**2. Email:**

```
security@parser-2gis.local
```

**3. Что включить в отчёт:**

- Тип уязвимости (SQL injection, XSS, RCE, и т.д.)
- Шаги для воспроизведения
- Версия Parser2GIS
- Возможные последствия
- Предложения по исправлению (опционально)

## Улучшения безопасности (v2.1.6)

### Критические улучшения

#### 1. Защита от SQL Injection
- **Severity:** Critical
- **Описание:** Валидация SHA256 хешей для кэша
- **Улучшение:** Строгая проверка входных данных
- **Версии:** < 2.1.6

#### 2. Защита от XSS атак
- **Severity:** Critical
- **Описание:** Валидация JavaScript кода
- **Улучшение:** Логирование и проверка всех скриптов
- **Версии:** < 2.1.6

#### 3. Защита от SSRF
- **Severity:** High
- **Описание:** Блокировка localhost и private IP
- **Улучшение:** Валидация URL
- **Версии:** < 2.1.6

#### 4. Защита от Race Condition
- **Severity:** High
- **Описание:** Атомарные операции с файлами
- **Улучшение:** RLock с timeout
- **Версии:** < 2.1.6

### Улучшения надёжности

#### 5. Управление ресурсами
- **Severity:** Medium
- **Описание:** Гарантированная очистка файловых дескрипторов
- **Улучшение:** finally блоки
- **Версии:** < 2.1.6

#### 6. Предотвращение Deadlock
- **Severity:** Medium
- **Описание:** Защита от взаимной блокировки
- **Улучшение:** RLock и timeout
- **Версии:** < 2.1.6

#### 7. Обработка кодировок
- **Severity:** Low
- **Описание:** Корректная обработка Unicode
- **Улучшение:** Fallback на альтернативные кодировки
- **Версии:** < 2.1.6

## Рекомендации по безопасности

### Для пользователей

**1. Регулярное обновление:**
```bash
pip install --upgrade parser-2gis
```

**2. Виртуальное окружение:**
```bash
python3 -m venv venv
source venv/bin/activate
pip install parser-2gis
```

**3. Переменные окружения:**
```bash
export GITHUB_TOKEN="your_token"
export PARSER_LOG_LEVEL="INFO"
```

**4. Разумные лимиты:**
```bash
parser-2gis --parser.max-records 1000
```

**5. Headless режим на серверах:**
```bash
parser-2gis --chrome.headless yes
```

### Для разработчиков

**1. Проверка зависимостей:**
```bash
pip audit
pip-review --auto
```

**2. Тестирование:**
```bash
pytest tests/ -v
```

**3. Pre-commit хуки:**
```bash
pre-commit install
pre-commit run --all-files
```

**4. Сканирование кода:**
```bash
bandit -r parser_2gis/
safety check
```
